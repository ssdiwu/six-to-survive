## 一、项目定位

浏览器 2D 弹幕 Roguelike 小游戏。玩家扮演打工人，钉在工位上，四面八方的飞书消息涌来，你靠疯狂敲键盘打字清活、甩锅反弹、摸鱼喘气，撑到 18:00 下班就算赢。

**测评用途**：豆包 Seed-Evolving 模型在 TRAE 中一次成型交付完整可玩网页游戏，体现生产级 Coding 能力（一次成型、表驱动、工程质量高、自主完成 UI/动效）。视频传播点：打工人共鸣 + 操作即叙事（按键=打字）+ 飞书生态梗。

## 二、核心玩法循环

### 一句话循环
**从 9:00 开始，飞书消息从四面八方涌来 → 你狂敲键盘打字射弹清活 → 每清一波三选一词条升级 → 靠摸鱼/甩锅应对压力 → 撑到 18:00 下班**

### 详细流程
```
启动游戏（标题页 → 开始）
  ↓
9:00 开局：玩家在屏幕中央工位圈，HP100/SAN100/精力100/KPI50
  ↓
飞书消息从屏幕四边持续涌入（波次随时间增强）
  ↓
玩家操作：
  ├─ WASD/方向键：在工位圈内小范围移动（躲弹）
  ├─ 任意字母/数字键：打出一个字符弹，朝最近敌人飞行造成伤害
  ├─ 空格（长按）：摸鱼（隐身回血，时间停走，KPI缓慢掉）
  └─ Shift（点按）：甩锅（全屏弹幕反弹为友方穿透弹，3秒不能攻击，反弹弹撞屏后回飞一次）
  ↓
资源博弈：
  ├─ 敲键 → 输出高，但精力掉
  ├─ 被弹幕击中 → HP掉、SAN掉
  ├─ 摸鱼 → SAN/精力回，但KPI掉、时间停走
  ├─ 甩锅 → 全屏清弹，但SAN掉（亏心）+ 锅会回飞
  └─ KPI低 → 触发"领导约谈"精英事件
  ↓
每清掉 N 个敌人（约每 30-45 秒一次）→ 三选一词条升级（被动为主）
  ↓
特殊事件：
  ├─ 老板查岗：探照灯扫过半屏，期间不能摸鱼/甩锅
  ├─ 飞书会议邀请：慢速大块，撞中硬直1秒
  ├─ 飞书文档@评论：中速弹，贴边停留持续放小弹
  ├─ 飞书审批待办：贴边持续掉精力
  └─ 同事@问问题：普通弹，被甩锅后变友方帮打
  ↓
17:30 Boss：周五紧急拉群（@全员）——弹幕洪水 + 中央群消息横幅压下
  ↓
18:00 → 胜利结算页（CI报告风 / 下班报告卡）
HP=0 或 SAN=0 → 失败结算（"加班猝死"/"当场崩溃"）
```

## 三、视角与地图

### 视角
- **斜 45° 俯视**（伪 3D，Canvas 2D 透视缩放实现）
- 屏幕中央是**工位**：一个方形/圆形区域（大约屏幕宽度的 25%），玩家在此内 WASD 移动
- 工位圈之外是"办公区虚空"——玩家不能移出工位圈（被钉在工位上）
- 敌人从屏幕四个边缘向中央工位圈飞入，有**透视缩放**（远小近大，制造纵深感）

### 视觉风格
- **暗色背景**（`#0d1117` GitHub Dark 基调）+ **飞书蓝**点缀（`#3370ff` 飞书品牌色）
- 飞书消息是**白色气泡弹**（带飞书 logo 或简化的头像方块）
- 玩家角色是一个打工人 emoji/简化头像（居中显示），被击中表情变化（😊→😰→😵）
- 伤害数字是打字机字体弹出，击杀有"✓已读"标记
- 屏幕顶部时间条：9:00 → 18:00，颜色渐变（早晨蓝→中午黄→傍晚红）

## 四、操控方案

| 键 | 行为 | 说明 |
|---|---|---|
| 任意字母/数字/符号键 | 打出一个字符弹 | 朝最近敌人飞行，伤害=基础伤害+词条加成，字符就是按键本身（随机显示A-Z/0-9/标点），视觉是飞书消息气泡颜色 |
| WASD / 方向键 | 在工位圈内移动 | 移动范围限制在工位圈（不出圈） |
| 空格（长按） | 摸鱼 | 见"主动技能"章节 |
| Shift（点按） | 甩锅 | 见"主动技能"章节 |
| P / Esc | 暂停 | 暂停菜单 |

**操作密度设计**：主操作就是疯狂敲键盘（高频、低决策），主动技能只给 2 个（低频、高决策）。不忙、不累、但有节奏感。

## 五、资源系统

### 5.1 四条资源

| 资源 | 初始值 | 含义 | 怎么涨 | 怎么掉 | 归零/满额后果 |
|---|---:|---|---|---|---|
| ❤️ HP | 100 | 生命值 | 不自动回（靠词条/摸鱼少量回） | 被弹幕撞中、老板扫到摸鱼、会议硬直期间被撞 | 归零 = 加班猝死，GG |
| 😰 SAN | 100 | 精神值 | 摸鱼回 | 被飞书@、会议硬直、甩锅后、KPI低领导约谈 | 归零 = 精神崩溃离场，GG；SAN<30 时屏幕扭曲、字符弹变成乱码 |
| ⚡ 精力 | 100 | 体力/打字速度 | 自然缓慢回、摸鱼回更快 | 每敲一键耗 0.5 点、被审批待办持续消耗 | 精力>80 攻速上限 100%；精力<30 攻速上限 -50%；精力=0 不能打字（只能走位/甩锅/摸鱼） |
| 📊 KPI | 50 | 业绩压力 | 每杀一个敌人涨 1-3 点 | 摸鱼时每秒掉 2 点 | KPI<20 触发「领导约谈」精英事件；KPI>80 触发「优秀员工」攻速 buff |

### 5.2 时间
- 屏幕顶部时间条：9:00 → 18:00，总时长约 **10-12 分钟**（玩家真实时间）
- 时间每秒推进约 45 秒游戏时间（可调，确保单局 10-12 分钟）
- 摸鱼期间**时间停走**（摸鱼不算干活，不算在下班倒计时里）
- 甩锅、暂停、升级选择期间时间继续走（但敌人停止刷出）

### 5.3 资源博弈核心钩子
```
狂敲键盘 → 清弹快 → 但精力掉 → 敲不动 → 弹幕堆 → HP危险
摸鱼 → SAN/精力回 → 但KPI掉 → 领导约谈（全屏伤害翻倍）
甩锅 → 全屏清弹 → 但SAN掉 + 锅回飞要清 + 3秒不能打
走位躲 → 省HP/SAN → 但DPS不够 → 敌人堆死
```
没有最优解，玩家必须在"拼命→burnout"和"摸鱼→被催"之间动态平衡。

## 六、敌人设计（飞书生态）

所有敌人从屏幕四边直飞中央（**不追踪、不拐弯**），有不同速度、体积、血量、行为。统一用飞书消息气泡视觉（白色气泡+头像色块+小图标区分类型）。

### 6.1 普通敌人

| 敌人 | 外观标识 | 速度 | HP | 伤害 | 特殊行为 | 隐喻 |
|---|---|---|---|---|---|---|
| **飞书 IM 消息** | 白色小气泡，蓝色头像 | 快 | 1 | 5 | 数量多，四面八方齐发 | 日常消息轰炸 |
| **飞书会议邀请** | 大块气泡，紫色+日历图标 | 慢 | 4 | 0（硬直） | 撞中玩家**硬直1秒**（不能敲键/甩锅），期间被其他弹打中双倍伤害 | 长会打断干活 |
| **飞书文档 @评论** | 黄色气泡，@图标 | 中 | 3 | 8 | 飞到工位边缘后**停驻**，每 2 秒释放一个小弹，必须优先敲掉本体 | "这里改一下"的持续需求 |
| **飞书审批待办** | 灰色气泡，对勾图标 | 慢 | 2 | 0（消耗） | 飞到屏幕边缘贴住，**每秒扣玩家 2 点精力**，贴满 3 个玩家就敲不动了，必须敲掉 | 流程卡你 |
| **同事@你问问题** | 绿色气泡，问号图标 | 中 | 2 | 6 | 普通弹；**被甩锅反弹后变成友方穿透弹**（帮你打其他敌人） | 同事的锅 |
| **飞书 OKR/季末对齐** | 红色大气泡，靶心图标 | 慢 | 8 | 15 | 精英怪，血厚伤害高，每 2 分钟刷新一只 | 周期评审压力 |

### 6.2 精英事件：老板查岗
- **触发时机**：KPI < 20 时高概率触发；游戏时间 11:30、14:30、16:30 固定触发
- **表现**：屏幕顶部出现一个老板头像（剪影），**像探照灯一样从一侧扫到另一侧**（约 4 秒扫完）
- **效果**：
  - 扫过的区域（半屏）玩家**不能摸鱼、不能甩锅**（按键弹出"👀 老板看着呢"提示）
  - 扫过期间所有飞进来的弹幕**伤害翻倍**
  - 摸鱼状态下被扫到直接扣 **40 HP**
- **应对**：只能硬敲扛过去，或者趁扫过前先摸鱼回满 SAN/精力

| 13:30-14:30 | 高 | 加入文档@评论 |
| 14:30 | - | 第二次老板查岗 + 第一只 OKR |
| 14:30-16:00 | 高 | 全类型混合刷 |
| 16:00-17:30 | 很高 | 多只 OKR 同时刷 |

## 七、主动技能

### 7.1 空格（长按）：摸鱼
- **效果**：玩家半透明隐身，所有弹幕**穿过不造成伤害**；每秒回 SAN 8 点、精力 12 点
- **代价**：
  - 时间**完全停走**（下班倒计时不动，摸鱼不算干活）
  - KPI 每秒掉 2 点
  - 松开摸鱼后有 **1.5 秒"回到状态"硬直**（攻速减半）
- **使用策略**：
  - HP/SAN 危险时的紧急喘气孔
  - 精力耗尽敲不动时回蓝
  - 不能在老板扫过期间用（被抓扣 40HP）
- **视觉**：角色变成咸鱼 emoji 🐟，背景变暗，屏幕上飘 zZZ~

### 7.2 Shift（点按）：甩锅
- **效果**：按下瞬间，屏幕上**所有朝向玩家的弹幕原地反向反弹**，变成友方穿透弹（伤害 = 原伤害 × 1.5），一路朝屏幕外飞，清掉路径上的所有敌人
- **代价**：
  - 反弹出去的弹幕撞到屏幕边缘后**沿原路径再飞回来一次**（回旋锅），变成敌对弹（玩家必须走位或敲掉）
  - 甩锅后 **3 秒不能敲键**（手在忙甩锅）
  - 每次甩锅扣 SAN 5 点（亏心事）
  - CD 5 秒
- **使用策略**：
  - 弹满屏时的清屏大招
  - 反弹同事@问问题的弹后变友方帮打
  - 但飞回来的回旋锅要接住（走位或敲掉）
- **视觉**：所有弹幕瞬间转 180°，每个贴一个 🍳 小锅 emoji 拖尾

## 八、被动词条升级系统

每击杀约 15 个敌人（约 30-45 秒）触发一次升级：**屏幕暂停 → 3 张随机词条卡 → 玩家点选 1 张（数字键 1/2/3 选择）**。

词条分 4 级稀有度：Common（白）/ Rare（蓝）/ Epic（紫）/ Legendary（金），前期易出白，后期随等级增加紫/金概率。

### 词条设计原则
- **80% 被动，20% 改玩法规则**（少量质变词条）
- 数值型 + 机制型混合，不只是"攻击力+10%"
- 要能形成 2-3 套 build 路线（打字流速射/甩锅流清屏/摸鱼流苟活）
- 具体词条池见下方 v0.1 列表（待平衡）

## 九、Boss 战：周五 17:30 紧急拉群

### 触发
游戏时间到达 17:30，屏幕渐红，群聊提示音响起，全屏弹出：
> 🚨 **紧急需求群** | 产品大拿: "@全员 今天下班前改完"

### Boss 机制
Boss 不是一个怪物，是**一个持续 3 分钟的生存阶段**（17:30 → 18:00），玩家需要撑过弹幕洪水 + 群消息横幅。

#### 阶段一：群消息轰炸（17:30-17:45）
- 四个方向同时涌入 IM 消息（密度是普通波次的 3 倍）
- 中央顶部挂一条红色群消息横幅**缓慢向下压**（像血条，占屏幕高度的 0%→40%）
- 横幅压到的位置玩家不能待（被压中持续掉血），但不致死，逼玩家在工位圈下半部走位

#### 阶段二：质疑弹幕（17:45-17:55）
- 加入特殊弹：**质疑弹**（蓝色，带文字）：
  - "这个逻辑不通" — 高速小弹
  - "明天能上吗" — 追踪形状但直飞（视觉吓人实际直飞）
  - "PM 改主意了" — 大号慢速弹，撞中清掉玩家一个被动词条
- 老板头像持续在屏幕上方徘徊（不扫动，但持续压视线），期间不能摸鱼

#### 阶段三：最后冲刺（17:55-18:00）
- 所有类型敌人混合齐刷
- 玩家攻速自动 +30%（deadline buff）
- 屏幕倒计时 "⏰ 距离下班还有 XX 秒"
- 撑到 18:00 → 胜利

### 胜利结算
屏幕渐绿，时间停在 18:00，弹出下班报告卡：
```
✓ 下班成功！
本局时长：11:23
击杀消息：342 条
甩锅次数：7 次（回旋锅击杀：12）
摸鱼时长：2:18
最高连击：47
获得词条：11 个

[下班]  [再来一局]  [分享战绩]
```
彩蛋：你发送了一条飞书消息——"我先下班了，周末不看飞书 👋"，然后飞书窗口关闭。

### 失败结算

## 十一、工程约束

### 交付形态
- **单文件 `index.html`**：HTML + CSS + JS 全部内联
- 零构建工具、零 npm 依赖、零本地服务器要求
- 浏览器双击打开即可玩
- 最终体积控制在 100KB 以内（不含 CDN 字体）

### 技术栈
- Canvas 2D API（不使用 three.js / PixiJS / WebGL）
- Vanilla JS（ES2020+），不用 TypeScript/框架
- Tailwind CSS **不使用**（单文件手写 CSS 足够，避免 CDN 依赖）
- 可选 Google Fonts CDN 引入 JetBrains Mono，必须有系统字体兜底
- 无音频依赖（可选 Web Audio API 生成简单打字音/提示音）

### 代码组织（index.html 内分区注释）
```js
// === 配置常量 ===  （敌人表、词条表、波次表、颜色、数值参数）
// === 游戏状态 ===  （player, enemies, bullets, particles, time, runState）
// === 工具函数 ===  （RNG, 向量, 碰撞检测, 随机选择）
// === 引擎核心 ===  （spawnEnemy, fireBullet, applySkill, applyUpgrade, tick）
// === 渲染层 ===   （drawPlayer, drawEnemies, drawBullets, drawHUD, drawBoss）
// === 输入处理 ===  （keydown/keyup, 按键状态映射）
// === 主循环 ===   （requestAnimationFrame, update + render）
```

### 数据驱动
所有敌人、词条、波次、技能参数都是 JS 对象/数组常量，可直接修改调参：

```js
};

  // ...

```

## 十四、风险与兜底

| 风险点 | 兜底砍法 |
|---|---|
| 斜 45° 透视实现复杂/翻车 | 改成纯俯视角（无透视），敌人远小近大通过 size 缩放简单模拟 |
| 任意键输入监听与浏览器快捷键冲突 | 监听 `keydown` 但阻止空格/方向键默认滚动；实在不行改"点击鼠标=攻击"备用方案 |
| 词条数值平衡差（太难/太易） | 用表驱动参数，开发末期调 ENEMY/BOSS 血量和伤害系数；保底所有敌人 HP -20% |
| Boss 三阶段代码复杂/崩 | 砍成单阶段（全弹幕洪水），去掉横幅压迫和质疑弹 |
| SAN 低视觉扭曲性能差 | 砍成简单色差（画面偏红+边缘暗化），不做噪点/乱码 |
| 甩锅反弹逻辑 bug（反弹弹不消失/无限循环） | 加一个"反弹弹最大反弹次数=1"的硬上限，防止无限弹 |
| 单文件体积超限 | 先拆 scripts/ 开发，最后构建时用简单正则合并内联 |
| 时间紧张来不及全部做完 | MVP 保核心循环：打字+WASD+IM消息+甩锅+升级+Boss单阶段，其他敌人类型/词条砍半 |

# 0002 · 纯俯视角替代斜 45° 透视

- 状态：已采纳
- 日期：2026-07-16

## 背景

PRD v0.1 设定为"斜 45° 俯视（伪 3D）"，通过 Canvas 2D 透视缩放实现纵深感：敌人从屏幕四边飞入时远小近大，工位以透视方形呈现。

该设定是在玩家有 WASD 移动的前提下提出的——透视帮助玩家判断远近、辅助走位。

## 决策

**采用纯俯视角（无透视缩放），不做斜 45° 伪 3D 效果。**

- 画面为二维平面坐标系，敌人位置和大小直接对应碰撞判定
- 工位为屏幕中央正方形（非透视梯形），玩家角色在其中心
- 弹幕从屏幕四边外生成，沿直线飞向中心，飞行过程中尺寸不变
- 纵深感靠弹幕密度、颜色层级和粒子演出制造，不靠透视

## 权衡

**选纯俯视的理由：**

- **弹幕可读性**：固定不动后玩家依赖视觉判断弹幕间距和类型，纯俯视下"看到的位置 = 碰撞位置"，不会出现"看起来远但判定已到"的偏差
- **实现简单**：不需要透视投影坐标变换、缩放插值；碰撞直接用屏幕坐标，AI 一次成型率高
- **透视在无移动时是花活**：没有走位就没有远近判断需求，透视缩放变成纯装饰，引入额外复杂度不划算

**放弃的东西：**

- 失去"四面八方压过来"的透视纵深感
- 工位区域的 3D 质感需要靠视觉细节（边框、阴影、角色演出）弥补

## 影响

- 渲染管线：所有实体使用屏幕直角坐标，无投影变换
- 碰撞检测：圆形/矩形碰撞即可，不需要透视校正
- 工位视觉：画一个正放正方形 + 边框 + 内部背景色区分
- Boss 演出：如果后续需要压迫感可在 17:30 做镜头倾斜特效，但基础视角保持纯俯视

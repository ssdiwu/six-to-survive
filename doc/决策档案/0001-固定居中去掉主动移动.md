# 0001 · 固定居中、去掉主动移动

- 状态：已采纳
- 日期：2026-07-16

## 背景

早期方案（见 PRD v0.1）设定玩家可通过 WASD / 方向键在工位圈内小范围移动，作为弹幕规避手段。

随后讨论中引入了"任意字母键 = 打字攻击"这一核心机制——玩家两只手都在键盘上疯狂敲键输出，操作本身就是"打字干活"的叙事。

## 决策

**玩家角色固定在屏幕中央工位正方形中心，不提供 WASD 或方向键主动移动控制。**

- 工位作为视觉场景存在（正方形区域内有坐姿/喝茶/踱步等演出动画），但玩家不能直接控制位置
- 规避伤害的手段从"走位"改为"主动道具"（摸鱼奶茶让敌人冻结、甩锅贴反弹弹幕、PPT 装甲护盾等）
- 弹幕进入工位正方形即判定击中

## 权衡

**选固定不动的理由：**

- **主题咬合**：被钉在工位上、四面八方来活只能硬接，比"在椅子上滑来滑去躲活"更贴打工人处境
- **操作纯粹**：所有可打印键都是打字攻击，不需要 carved out WASD 四个键；玩家按 W 不会疑惑"为什么我没打出弹"
- **决策转移**：规避方式从反应走位转为资源决策（什么时候摸鱼、什么时候甩锅），更贴合 Roguelike 构筑感

**放弃的东西：**

- 失去走位带来的微操乐趣
- 弹幕游戏的传统"躲"需要重新设计为技能/道具应对
- 部分敌人行为（追踪弹、需要走位的区域封锁）首版无法实现

## 影响

- 输入系统：WASD/方向键不绑定任何游戏行为（静默忽略）
- 碰撞：以工位正方形而非角色小圆为受击区域
- 敌人设计：所有敌人直线飞向屏幕中心，不做追踪/拐弯行为
- 角色演出：在工位正方形内根据玩家状态切换姿势/位置（自动，非玩家控制）
- Boss：原 PRD 中"横幅压下逼走位"的机制首版砍掉，改为下班准备期密度 rush
